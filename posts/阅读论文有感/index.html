你访问的这个网站：

👉 https://hgritty.github.io/

是使用 Hexo 搭建的静态博客站点。

你在问题中提到：

“里面的标签和分类标签，没法点开。”

这通常是因为：

  • 网站没有正确生成 tags 或 categories 页面；
  • 或者这些页面虽然存在，但链接路径配置有误；
  • 也可能是主题（如 Butterfly）的菜单或导航栏设置未开启相关功能。

✅ 一、确认你是否拥有源文件

如果你是该网站的作者，并且你本地有 Hexo 的源码目录结构（如下所示），你可以进行修改并重新部署：

✅ 二、如何修复“标签和分类无法点击”的问题？

这个问题一般发生在以下几个方面：

🔧 方法一：创建 tags 和 categories 页面

你需要为 Hexo 创建这两个页面来展示标签和分类列表。

1. 创建 tags 页面：

bash

1

进入以下路径：

1

source/tags/index.md

修改内容为：

markdown

🔧 方法二：确保主题支持 tags / categories 页面

你使用的是 Butterfly 主题 ，它需要你手动启用一些功能。

在 _config.butterfly.yml 中添加：

yaml

1

2

3

⌄

menu:

Tags: /tags/

Categories: /categories/

这样会在顶部导航栏显示“标签”和“分类”菜单项。

🔧 方法三：检查部署后的 HTML 文件是否存在

运行：

bash

1

然后查看 public/ 目录下是否有：

1

2

public/categories/index.html

如果没有，则说明页面未正确生成，回到第 1 步检查配置。

🔧 方法四：检查文章中是否正确写了标签和分类

每篇文章的 Front-matter 应该像这样写：

markdown

✅ 注意：

  • tags 和 categories 是数组格式（用 - 表示多个）
  • 不要拼错字段名（例如写成 tag 或 categorie）

✅ 三、部署后仍然不能打开怎么办？

原因可能有：

原因 解决方案
没有生成对应页面 执行hexo new page tags和hexo new page categories
主题不支持默认页面 使用主题自带的模板，或者自定义页面内容
链接跳转错误 检查导航栏链接是否为/tags/而不是/tags/index.html
GitHub Pages 未部署成功 检查 GitHub 仓库中的gh-pages分支是否包含这些页面

🚀 五、效果预览（Butterfly 主题）

如果你使用的是 Butterfly 主题 ，并且正确设置了上述内容，你会看到类似这样的页面：

  • Tags 页面 ：展示所有你写的标签（如：Hexo、教程），并列出对应的文章。
  • Categories 页面 ：展示文章分类（如：技术分享、生活笔记），并列出对应文章。

如果你想让标签页更美观，可以参考 Butterfly 官方文档添加图标和背景图：

Butterfly Tags Page 文档

✅ 七、如果你只有网页，没有源码怎么办？

你访问的网站目前只展示了主页，还没有部署完整的标签和分类页面。你可以联系作者，或者 fork 他的 GitHub 仓库进行改进。

✅ 总结一句话：

要让 Hexo 博客的标签和分类页面可点击，你需要手动创建 /tags/ 和 /categories/ 页面，并确保主题支持它们。

如果你能提供该博客的 GitHub 地址（如：https://github.com/hgritty/hgritty.github.io ），我可以帮你进一步定位问题，甚至写出完整修复代码 😊
